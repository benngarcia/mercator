# Code Of Conduct

Mercator is infrastructure software. Public discussion should make the project
safer, clearer, and easier to evaluate without making contributors feel unsafe
or unwelcome.

## Scope

This code applies to Mercator project spaces, including GitHub issues, pull
requests, discussions, release notes, docs, and project-related review threads.
It also applies when someone represents the project in another public forum.

## Expected Behavior

- Be concrete, respectful, and technical.
- Assume good intent while still asking for evidence.
- Critique code, docs, designs, and operational claims, not people.
- Keep safety-sensitive reviews direct and specific.
- Respect private disclosure paths for security reports, credentials, customer
  data, hostnames, local identifiers, and unreleased downstream details.

## Unacceptable Behavior

- Harassment, threats, sustained disruption, or personal attacks.
- Discriminatory language or conduct.
- Sexualized comments, imagery, or attention.
- Publishing private information without explicit permission.
- Pressuring someone to disclose security details or private operational data in
  a public thread.
- Spam, impersonation, or bad-faith participation.

## Report A Conduct Concern

Do not post sensitive conduct reports publicly. Until the project has a
dedicated conduct contact, contact the maintainer through GitHub and ask for a
private channel. If the concern also involves a vulnerability or secret leak,
follow [SECURITY.md](SECURITY.md).

Useful reports include:

- the affected project space or link;
- the people involved;
- what happened and when;
- any relevant screenshots, logs, or prior moderation attempts;
- whether the report includes private security or customer information.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests;
block users; restrict participation; or move a discussion to a private channel.
The response should be proportionate to the impact and should prioritize
project safety, reporter safety, and a useful technical record.

Retaliation against someone who makes a good-faith conduct or security report
is not acceptable.
