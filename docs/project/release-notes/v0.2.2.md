# Mercator v0.2.2

Mercator `v0.2.2` is a pre-GA patch release. The HTTP API, CLI, and event
schemas are unchanged from `v0.2.1`; existing event logs and already-launched
containers are fully compatible.

## Container Image

The container image is published as `ghcr.io/benngarcia/mercator:0.2.2`
(also `:0.2` and `:latest`). Pin deployments by digest:

```
ghcr.io/benngarcia/mercator:0.2.2@sha256:29d23b43c57a558dfd0b3872404c260ca18f23599e7a8bdcee06aa05b3921de4
```

## What Changed

### Fixes

- Run advancement is serialized per run, so the background reconcile sweep can
  no longer race an in-flight launch for the same run. Previously a sweep tick
  and a client-driven advance could both step the same run concurrently and
  attempt a duplicate launch.
- The RunPod adapter honors the GPU and disk resources recorded in the
  scheduled offer instead of falling back to pod defaults, so a run lands with
  the resources the placement decision actually priced.

### Cleanup

- Unreachable runtime abstractions and unreachable console UI/demo machinery
  were deleted, and the launch docs were reworked around executable evidence.
  No behavior change.

## Install

Download the archive for your OS and architecture from the GitHub Release,
then download `checksums.txt` from the same release.

For Linux:

```sh
version=v0.2.2
os=linux
arch=amd64

curl -LO "https://github.com/benngarcia/mercator/releases/download/${version}/mercator_${version}_${os}_${arch}.tar.gz"
curl -LO "https://github.com/benngarcia/mercator/releases/download/${version}/checksums.txt"
sha256sum -c checksums.txt --ignore-missing
tar -xzf "mercator_${version}_${os}_${arch}.tar.gz"
sudo install "mercator_${version}_${os}_${arch}/mercator" /usr/local/bin/mercator
```

For macOS, set `os=darwin`, choose `arch=arm64` or `amd64`, and use:

```sh
shasum -a 256 -c checksums.txt --ignore-missing
```

Each archive contains `mercator`, `README.md`, `LICENSE`, and `NOTICE`.

## Upgrading From v0.2.1

Point the new binary at the same `MERCATOR_SQLITE_DSN`. The event log format,
public event schemas, and adapter wire contract are unchanged.

## Known Limitations

Read `docs/production/known-limitations.md` before relying on this release for
anything production-adjacent. Security issues should be reported privately
through `SECURITY.md`.
