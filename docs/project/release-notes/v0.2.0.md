# Mercator v0.2.0

Mercator `v0.2.0` is a pre-GA evaluation release focused on making the broker
simpler to reason about and less dependent on client polling. The HTTP API,
CLI, SDKs, and event schemas are unchanged from `v0.1.0`; existing event logs
and already-launched containers are fully compatible.

## What Changed

### Runs advance in the background

`serve` now drives open runs forward in a background reconcile loop, so a run
reaches its terminal state, records its outcome, and cleans up its container
even if no client ever polls it again. Previously a run only progressed when a
client hit create, `:wait`, or `:refresh`.

### One advance loop in the orchestrator

Run advancement is now a single reduce→step loop: every entry point (create,
cancel, report, poll, background reconcile) appends its fact events and funnels
through the same loop, which re-derives state from the event log on each
iteration. Two behavior fixes fall out of this:

- a poll after a failed cancel now completes the cancel instead of ignoring it;
- a poll after a reported exit finalizes the run even if the report-time
  advance was interrupted.

### Smaller, single-altitude codebase

Dormant subsystems were removed (an unused reconciler wrapper, projection
runner, latency estimator, and a cached-offers service whose write path was
never wired — `GET /v1/offers` always effectively served live adapter offers,
and now does so explicitly). Docker offer synthesis moved out of the entrypoint
into the Docker adapter. Net effect: ~650 fewer lines and four fewer internal
packages, with no behavior change.

### Fixes

- Connection event ids are scoped by workspace, so identical connection ids in
  different workspaces can no longer collide in the event log.

## Install

Download the archive for your OS and architecture from the GitHub Release,
then download `checksums.txt` from the same release.

For Linux:

```sh
version=v0.2.0
os=linux
arch=amd64

curl -LO "https://github.com/benngarcia/mercator/releases/download/${version}/mercator_${version}_${os}_${arch}.tar.gz"
curl -LO "https://github.com/benngarcia/mercator/releases/download/${version}/checksums.txt"
sha256sum -c checksums.txt --ignore-missing
tar -xzf "mercator_${version}_${os}_${arch}.tar.gz"
sudo install "mercator_${version}_${os}_${arch}/mercator" /usr/local/bin/mercator
```

For macOS, set `os=darwin`, choose `arch=arm64` or `amd64`, and use:

```sh
shasum -a 256 -c checksums.txt --ignore-missing
```

Each archive contains `mercator`, `README.md`, `LICENSE`, and `NOTICE`.

The container image is published as
`ghcr.io/benngarcia/mercator:0.2.0` (also `:0.2` and `:latest`).

## Upgrading From v0.1.0

Point the new binary at the same `MERCATOR_SQLITE_DSN`. The event log format,
public event schemas, and adapter wire contract (container labels, pod env) are
unchanged, so existing runs, connections, and externally launched objects are
picked up as-is.

## SDKs

SDK registry publishing remained deferred, so users installed the included
packages from a source checkout. They are preserved in the immutable release
tag:

- [TypeScript SDK at v0.2.0](https://github.com/benngarcia/mercator/tree/v0.2.0/sdk/typescript)
- [Python SDK at v0.2.0](https://github.com/benngarcia/mercator/tree/v0.2.0/sdk/python)
- [Ruby SDK at v0.2.0](https://github.com/benngarcia/mercator/tree/v0.2.0/sdk/ruby)

The current source tree no longer maintains these packages.

## Known Limitations

Read `docs/production/known-limitations.md` before relying on this release for
anything production-adjacent. Security issues should be reported privately
through `SECURITY.md`.
