# Mercator v0.5.4

This patch completes upgrades that passed through v0.5.2 before installing
v0.5.3. The SQLite migration now recognizes both original
`compute.run.placement_decided.v1` events and partially migrated
`compute.run.booking_decided.v1` events whose selected decision still lacks
Booking identity.

The same safety boundary applies: migration is automatic, transactional, and
idempotent for closed runs. Mercator fails startup explicitly rather than
inventing live Rental state for an open legacy run.
