# Mercator v0.5.2

This patch repairs upgrades from releases that stored
`compute.run.placement_decided.v1` events. Mercator now migrates those event
types to `compute.run.booking_decided.v1` when it opens the SQLite database, so
run listing and lookup continue to work after the durable placement vocabulary
changed to bookings in v0.5.0.

The event payload, identity, ordering, and stream version remain unchanged.
The migration is automatic and idempotent. Operators should retain their normal
pre-upgrade SQLite backup; no manual migration command is required.
