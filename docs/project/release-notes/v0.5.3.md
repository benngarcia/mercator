# Mercator v0.5.3

This patch completes the SQLite upgrade introduced in v0.5.2. Archived
`compute.run.placement_decided.v1` events now receive the Booking identity and
Rental identity required by the v0.5 event contract while their event type is
renamed.

The migration is automatic, transactional, and idempotent for closed runs.
Mercator fails startup explicitly if a database contains an open legacy run
because its pre-schedule event stream does not contain enough information to
reconstruct live Rental state safely. Finish or cancel those runs before
upgrading.
