Suggested labels: `good first issue`, `docs`, `console`, `launch`

## Problem

The README has a short fake-adapter demo and transcript, but the launch
scorecard includes a 75-100 second shot list that can show the full evaluation
path: quickstart, run creation, terminal run fields, console run list, decision
tab, public events, and docs map.

## Acceptance Criteria

- Follow the shot list in `docs/launch/open-source-readiness.md`.
- Keep the video free of private tokens, provider credentials, hostnames, local
  machine identifiers, and customer data.
- Add the selected video under `docs/assets/` or document why it should stay
  externally hosted.
- Include captions or a text transcript.
- Keep the existing short WebM/GIF demo in place.
- Update `docs/assets/README.md` with capture notes for the final artifact.
